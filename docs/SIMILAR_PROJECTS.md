# MeDF Hub と類似プロジェクトの比較

**バージョン**: 1.0  
**最終更新**: 2026年2月8日

---

## 概要

このドキュメントでは、MeDF Hubと同様の機能を持つIPFS関連プロジェクトを比較します。各プロジェクトの設計思想・技術スタック・ユースケースを整理することで、MeDF Hubの特徴と位置づけを明確にします。

## 各プロジェクトの詳細

### 1. OrbitDB

**概要**: サーバーレスな分散P2Pデータベース

**特徴**:

- **データモデル**: イベントログ・ドキュメント・キー・バリュー
- **CRDT**: Merkle-CRDTベースの競合フリー書き込み
- **同期**: Libp2p Pubsubによる自動同期
- **暗号化**: オプションの暗号化機能
- **ブラウザ対応**: ブラウザとNode.jsの両対応

**MeDF Hubとの違い**:

| 観点 | MeDF Hub | OrbitDB |
|-----|---------|---------|
| **設計目標** | 文書検証・議論 | 汎用分散DB |
| **ハッシュ検証** | RFC 8785準拠（JSON正規化） | Merkle-CRDT（操作ベース） |
| **コメント機能** | ✅ ブロック単位 | ❌ なし |
| **参照管理** | ✅ MEDF引用追跡 | ❌ なし |
| **中央集約性** | Pinataで公開 | 完全P2P |
| **用途** | 学術文書・議論 | 汎用データ同期 |

**適用シーン**:

- 完全分散型のアプリケーション（チャット・ゲーム・協調編集）
- 中央サーバーに依存しない設計が必須の場合
- リアルタイム同期が重要な場合

---

### 2. Ceramic Protocol

**概要**: ブロックチェーンアンカー付きの分散イベントストリーム

**特徴**:

- **イベントストリーム**: 署名済みイベントの追記型ログ
- **DID**: Decentralized Identifierベースの認証
- **ブロックチェーン**: Ethereumへのアンカリング（タイムスタンプ）
- **ComposeDB**: グラフデータベース層
- **スケーラビリティ**: 非同期・並列処理による高スループット

**MeDF Hubとの違い**:

| 観点 | MeDF Hub | Ceramic |
|-----|---------|---------|
| **タイムスタンプ** | データベース | Blockchain |
| **認証** | OAuth | DID |
| **コスト** | 低（Pinata） | 中（Blockchain） |
| **検証方式** | RFC 8785 | Merkle木 |
| **議論機能** | ✅ | ❌ |
| **用途** | 学術文書 | アイデンティティ・評判 |

**適用シーン**:

- ブロックチェーンのタイムスタンプが必須の場合
- 分散アイデンティティ・評判スコアの管理
- 完全な改ざん防止が必須（法的証拠）

---

### 3. Textile ThreadDB

**概要**: IPFSベースの分散ドキュメント指向DB（メンテナンス停止）

**特徴**:

- **スレッド**: 複数ユーザーによる協調編集
- **Merkle-CRDT**: 競合フリーな書き込み
- **暗号化**: エンドツーエンド暗号化
- **マルチライター**: 複数ユーザーの同時書き込み

**MeDF Hubとの違い**:

| 観点 | MeDF Hub | ThreadDB |
|-----|---------|----------|
| **ステータス** | 本番環境対応 | ⚠️ メンテナンス停止 |
| **文書フォーマット** | MeDF v0.2.1 | JSON |
| **議論機能** | ✅ | ❌ |
| **参照管理** | ✅ | ❌ |
| **用途** | 学術文書 | 協調編集 |

**注意**: Textile ThreadDBは2022年4月以降メンテナンスが停止しており、本番環境での利用は推奨されません。

## MeDF Hubの独自性

### 1. ブロック単位の議論機能

**特徴**: 文書の特定ブロックに対して、参照付きコメントを投稿可能

```
MEDF: document-id#block-id という形式で引用
↓
自動的にリンク化・参照グラフを構築
↓
スレッド型議論で学術的な検証プロセスを支援
```

**他プロジェクトにない理由**: 

- OrbitDB・Ceramic・ThreadDBは汎用DB設計で、文書構造を前提としない
- ブロック単位の議論は学術文書向けの特殊な要件

### 2. RFC 8785準拠のハッシュ検証

**特徴**: JSON正規化によるオフライン検証

```
RFC 8785 正規化
↓
SHA-256ハッシュ計算
↓
ブロック・ドキュメント単位での検証
↓
Web Crypto APIで完全オフライン対応
```

**他プロジェクトとの違い**:

| プロジェクト | 検証方式 |
|-----------|--------|
| MeDF Hub | RFC 8785 JSON正規化 |
| OrbitDB | Merkle-CRDT（操作ベース） |
| Ceramic | Merkle木 + Blockchain |
| ThreadDB | Merkle-CRDT（操作ベース） |

### 3. 参照トラッキング機能

**特徴**: MEDF引用の自動検出・リンク化・双方向追跡

```
コメント内の「MEDF: doc-id#block-id」を検出
↓
参照テーブルに自動登録
↓
「この文書を参照している文書」を逆引き可能
↓
学術的な引用グラフを構築
```

**用途**:

- 論文の引用関係を可視化
- 関連研究の発見
- 学術的な信頼性の評価

### 4. エクスポート機能

**特徴**: HTML・PDF・JSON形式での出力

```
HTML: メタデータ・TOC・ハッシュ表示付き
PDF: ブラウザ印刷ダイアログ経由
JSON: MeDF形式でのダウンロード
```

**他プロジェクトにない理由**: 

- 学術文書の配布・印刷が想定されるユースケース
- 汎用DBは出力形式を提供しない

## ユースケース別推奨プロジェクト

### 学術文書の共有・議論

**推奨**: **MeDF Hub**

**理由**:

- ブロック単位のコメント・議論機能
- 参照トラッキングで引用関係を管理
- エクスポート機能で配布可能
- RFC 8785準拠で検証可能

### 完全分散型のリアルタイム協調編集

**推奨**: **OrbitDB**

**理由**:

- 完全P2P設計で中央サーバー不要
- Merkle-CRDTで競合フリー
- リアルタイム同期
- ブラウザ・Node.js両対応

### ブロックチェーンタイムスタンプが必須

**推奨**: **Ceramic Protocol**

**理由**:

- Ethereumへのアンカリング
- 改ざん防止の法的証拠
- DIDベースの認証
- 高スケーラビリティ

### 分散アイデンティティ・評判管理

**推奨**: **Ceramic Protocol**

**理由**:

- ComposeDBでグラフデータベース化
- DIDによる分散アイデンティティ
- 評判スコア・信頼スコアの管理

## 技術的な比較

### ハッシュ検証方式

**MeDF Hub**:

```
入力: MeDF JSON
  ↓
RFC 8785 正規化
  ↓
SHA-256 ハッシュ
  ↓
ブロック・ドキュメント単位での検証
```

**OrbitDB**:

```
操作ログ（OpLog）
  ↓
Merkle-CRDT
  ↓
競合フリーな状態変化
  ↓
イベント単位での検証
```

**Ceramic**:

```
イベント
  ↓
Merkle木
  ↓
Blockchain アンカー
  ↓
グローバルタイムスタンプ
```

### 同期メカニズム

| プロジェクト | 同期方式 | リアルタイム性 | 一貫性 |
|-----------|--------|-------------|------|
| MeDF Hub | 中央DB + Pinata | 中程度 | 強い一貫性 |
| OrbitDB | Libp2p Pubsub | 高い | 結果整合性 |
| Ceramic | Blockchain | 低い | 強い一貫性 |
| ThreadDB | Libp2p Pubsub | 高い | 結果整合性 |

## 今後の発展方向

### MeDF Hubの拡張案

1. **完全P2P化**: OrbitDBのようなLibp2p統合で完全分散化
2. **Blockchain統合**: Ceramicのようなタイムスタンプ機能
3. **マルチシグ**: 複数ユーザーによる署名ベース検証
4. **バージョン管理の高度化**: Git的な分岐・マージ機能

### 他プロジェクトとの連携

**可能性**:

- OrbitDB + MeDF Hub: P2P同期 + 議論機能
- Ceramic + MeDF Hub: Blockchainタイムスタンプ + 議論機能
- Textile + MeDF Hub: 暗号化 + 議論機能

## 結論

### MeDF Hubの位置づけ

MeDF Hubは、**学術文書の共有・検証・議論に特化した中程度の分散性を持つプラットフォーム**です。

**強み**:

- ✅ ブロック単位の議論機能（他にない）
- ✅ RFC 8785準拠の検証（オフライン対応）
- ✅ 参照トラッキング（学術的）
- ✅ エクスポート機能（配布向け）
- ✅ 学習曲線が低い（Web標準）

**トレードオフ**:

- ❌ 完全P2P ではない（Pinata依存）
- ❌ Blockchainタイムスタンプなし
- ❌ 高度な暗号化機能なし

### 推奨される用途

1. **学術論文の公開・議論**: ✅ 最適
2. **研究チームの協調編集**: ✅ 適切
3. **知識ベースの構築**: ✅ 適切
4. **完全分散型アプリ**: ❌ OrbitDB推奨
5. **法的証拠が必須**: ❌ Ceramic推奨
